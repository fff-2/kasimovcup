2하드웨어 구성

나노 2대 체제입니다. TCS34725 컬러센서는 I2C 주소(0x29)가 고정이라 한 버스에 2개를 못 달기 때문에, 앞 센서는 메인 나노가 직접 읽고 뒤 센서는 전담 나노가 읽어 감지 결과만 디지털 신호 한 가닥으로 넘깁니다.

3전술 — 경계선을 도는 사냥꾼

정면 힘싸움을 피하고, 경계선을 따라 돌다가 만나는 순간 먼저 보고, 먼저 꺾어서, 비스듬히 밀어 장외시키는 전술입니다.

  1. 기습 출발
    시작하자마자 45° 꺾어 직진 — 상대의 첫 정면 돌격을 흘려보낸다
  2. 경계선 선회
    앞 컬러센서가 빨간 선을 잡으면 선을 따라 원을 그리며 순찰
  3. 조우 → 선제 꺾기 → 밀어내기
    초음파로 상대를 먼저 감지하고 살짝 안쪽으로 꺾어 상대 측면을 잡는다. 상대는 이미 경계선 근처 — 조금만 밀어도 장외

조우는 딱 두 가지 형태로 옵니다

A. 서로 반대로 돌 때 — 정면에서 마주침. 앞 초음파 30cm에서 잡고 안쪽으로 150ms 선회 후 전속 밀기.

B. 같은 방향으로 돌 때 — 속도 차로 추격전. 뒤 초음파에 잡히면 가속하며 안쪽으로 파고들고, 상대가 바깥으로 지나치면 그대로 측면을 민다. 서로 영영 안 만나면 8초 후 중앙을 가로질러 판을 깬다.

이 전술의 안전핀: 라인이 항상 공격보다 우선 경계선 근처에서 싸우는 전술이라 자기 장외가 가장 큰 리스크입니다. 그래서 코드는 밀기 도중이라도 앞에 빨간 선이 보이면 무조건 제동→후진→복귀합니다. 7cm 선 두께 덕분에 감지 후 실제 장외까지 여유가 있습니다.

6경기 당일 체크리스트

경기 전 준비

  • 두 스케치 모두 DEBUG 0으로 업로드했는가 (시리얼 출력이 루프를 느리게 함)
  • 업로드 안 되면 보드 설정을 ATmega328P (Old Bootloader)로 (호환 나노 대부분)
  • 배선 3대 실수 점검: GND 공통 / A6 풀다운 저항 / 컬러센서 LED ON
  • 바퀴 방향, 45° 회전량, 라인 감지, 30cm 공격 트리거 실측 확인 (5절의 ★ 순서)
  • 새 9V 배터리 경기 수만큼 준비

매 경기 루틴

  • 배터리 새것으로 교체 (전압 낮으면 45° 회전부터 틀어짐)
  • 로봇을 흰 바닥 위에 올려둔 상태로 전원 인가 — 컬러 나노가 2초 뒤 자동 캘리브레이션 (LED 켜졌다 꺼지면 완료)
  • 심판 신호에 시작 버튼 — 메인 나노가 0.5초 캘리브레이션 후 바로 기동
  • 작전타임은 -3점. 라인 감지 불량·모터 역회전 같은 치명 문제에만 사용
하프타임(2분)에 할 수 있는 것 HW/SW 수정이 허용되므로, 전반에 본 상대 성향에 맞춰 상수를 조정할 수 있습니다. 대표 카드: 상대와 같은 방향으로 돌아 교착이면 CIRCLE_DIR 반전 또는 BORED_MS 단축, 상대가 저돌적이면 ATTACK_DIST_CM을 늘려 더 일찍 대응.
